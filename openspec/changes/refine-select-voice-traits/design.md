## Context

当前 `select voice` 页面使用固定的声音特质分类与分类映射，`童声` 仍在可选列表中，但该业务分类已不再需要。与此同时，产品需要新增 `专业播音` 与 `特色人物` 两个分类，并将指定声音（白岩松、罗翔、沈逸、马云、杨幂、周杰伦、trump罗翔）按分类准确呈现。  
该变更主要涉及声音分类配置、分类渲染、分类筛选和默认选中策略，目标是在不改变用户已有其他选音流程的前提下完成结构升级。

## Goals / Non-Goals

**Goals:**
- 删除 `童声` 分类，使其不再在 UI 中显示也不再参与筛选。
- 新增 `专业播音`、`特色人物` 分类并按要求挂载对应声音条目。
- 保证分类和声音映射由单一配置源驱动，避免 UI 文案与数据源脱节。
- 保持现有页面交互行为稳定（切换分类、加载声音、选择声音）。

**Non-Goals:**
- 不调整声音合成底层能力，不修改音频模型本身。
- 不新增推荐算法、排序策略或个性化逻辑。
- 不在本次变更中扩展更多分类，仅覆盖指定分类和声音。

## Decisions

### 1) 采用配置驱动的分类-声音映射
将声音特质分类及成员维护在统一配置结构中（常量表或服务端响应结构），页面仅消费该结构渲染分类与声音列表。  
这样可以避免分类文案、筛选枚举、显示列表分散在多个文件，降低后续继续增删分类的维护成本。

候选方案对比：
- 方案 A（采纳）：单一映射配置源 + UI 按配置渲染。
- 方案 B（未采纳）：在 UI 各处硬编码分类与声音过滤条件。  
未采纳原因：易出现遗漏，且后续扩展成本高。

### 2) 对移除分类采取“完全下线”策略
`童声` 不仅从标签展示移除，也从筛选与数据映射层剔除，避免隐藏入口但仍可被旧参数命中。  
若存在历史持久化值（如本地状态里仍为 `童声`），在初始化时回退到默认分类（例如首个有效分类）。

候选方案对比：
- 方案 A（采纳）：展示层与数据层同步移除。
- 方案 B（未采纳）：只在 UI 隐藏，底层保留。
未采纳原因：可能引发隐性状态 bug，且不利于数据一致性。

### 3) 保持增量改动，不重构页面结构
仅替换分类数据与筛选入口，不改变 `select voice` 页面组件层级与交互流程。  
这样能把风险收敛在配置和映射逻辑，降低回归范围。

## Risks / Trade-offs

- [风险] 历史缓存/持久化状态仍引用 `童声`，导致页面首次进入异常。  
  [缓解] 在状态恢复阶段增加合法性校验，不合法值自动回退默认分类。

- [风险] 新增声音名称与真实 voice id 映射不一致，出现“有标签无声音”或“点选失败”。  
  [缓解] 在映射表中使用稳定 id，并增加启动时或单测中的映射完整性校验。

- [风险] 分类顺序预期不一致影响产品验收。  
  [缓解] 在配置中显式定义排序顺序，并加入 UI 快照/断言测试。

## Migration Plan

1. 更新分类与声音映射配置：删除 `童声`，新增 `专业播音`、`特色人物` 及指定声音集合。  
2. 更新分类初始化逻辑：若命中废弃分类值则回退默认。  
3. 执行页面联调与回归测试，确认分类切换、声音加载、声音选择正常。  
4. 如需回滚，仅恢复上一版分类映射配置和对应常量定义即可。
