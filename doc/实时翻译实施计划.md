## 模块详情

### 模块 1：translation.yml

**文件**：`apps/moxin-voice/dataflow/translation.yml`

节点：`moxin-audio-input` → `dora-qwen3-asr` → `dora-qwen3-translator` → `moxin-translation-listener`

---

### 模块 2：dora-qwen3-translator

**位置**：`node-hub/dora-qwen3-translator/`

核心逻辑：
- 加载 `qwen3-mlx` 模型（Qwen3-8B 4-bit，路径 `~/.OminiX/models/qwen3-8b-4bit`）
- 接收 ASR 转写文本（+ `src_lang`、`tgt_lang` metadata）
- 句法完整性校验（中文残句检测）
- 构建翻译 prompt（`/no_think` + 单次翻译指令）
- 流式生成 → token 级输出

---

### 模块 3：VAD 分句（含在模块 2）

在翻译节点内部或 `moxin-audio-input` 侧实现：
- 语言感知静音阈值：ZH=500ms、EN=350ms、FR=400ms
- 强制最大块时长：8s
- 中文句法完整性校验（`的`/连词结尾检测）

---

### 模块 4：translation-listener（Dora bridge）

**位置**：`moxin-dora-bridge/src/widgets/translation_listener.rs`

逻辑：
- Dynamic Dora 节点，订阅 `translator/source_text` 和 `translator/translation`
- 通过 `Cx::post_action(TranslationUpdate {...})` 推送到 UI 线程

---

### 模块 5：TranslationOverlay Widget

**位置**：`moxin-widgets/src/translation_overlay.rs`

组件：
- 工具栏（源语言/目标语言选择）
- 原文显示区（小字，暂定色）
- 译文显示区（大字，主体）
- 状态指示（就绪/识别中/翻译中）

---

### 模块 6：多窗口 App Shell

**文件**：`moxin-voice-shell/src/app.rs`

修改：在 `App` 的 `live_design!` 中增加 `translation_window = <Window> {...}`

---

### 模块 7：主界面控件

**文件**：`apps/moxin-voice/src/screen.rs`

新增：
- 页头"翻译"切换按钮
- 语言选择下拉（输入语言 / 输出语言）
- 翻译 Dora 数据流的启动/停止逻辑

---

### 模块 8：模型下载脚本

**文件**：`models/model-manager/download_models.py`

新增 `--download qwen3-8b-4bit` 选项，下载到 `~/.OminiX/models/qwen3-8b-4bit/`
