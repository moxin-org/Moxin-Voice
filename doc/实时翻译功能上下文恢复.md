## 1. 功能概述

侧边栏点击"实时翻译"，切换到独立翻译页面：

**设置态（启动前）**：用户在页面内配置输入源、输入语言、目标语言、浮窗样式，点击"启动实时翻译"后：

1. 启动独立 Dora 数据流：`麦克风 → ASR → Qwen3 翻译 → bridge 节点`
2. 弹出独立 OS 浮窗（第二个 NSWindow），实时显示 ASR 原文 + 逐 token 译文
3. 翻译全程本地运行（Qwen3-8B-4bit，MLX，Apple Silicon），无云端依赖

**运行态（启动后）**：页面切换为监控视图，显示 CPU/内存/VAD 状态 + 实时滚动日志，"停止翻译"按钮返回设置态。

支持语言对：ZH/EN/FR/JA → 任意目标语言（当前 UI 提供 ZH/EN/JA/FR 选项）。

## 5. 关键实现细节

### 5.1 SharedDoraState 新增字段

文件：`moxin-dora-bridge/src/shared_state.rs`

```rust
pub translation: DirtyValue<Option<TranslationUpdate>>,
pub translation_window_visible: DirtyValue<bool>,
```

`TranslationUpdate`（在 `data.rs`）：
```rust
pub struct TranslationUpdate {
    pub source_text: String,        // ASR 原文
    pub translation: String,        // 累积译文（流式时为当前全部内容）
    pub is_complete: bool,          // true = 句子翻译完成
    pub prev_source_text: String,   // 上一完成句原文（防 DirtyValue 覆盖丢失）
    pub prev_translation: String,   // 上一完成句译文
}
```

> **设计说明**：`prev_source_text` / `prev_translation` 由 listener 维护，每次 `set()` 都携带。
> 解决的问题：`DirtyValue` 是 last-write-wins，如果句子 A 的 `is_complete=true` 和句子 B 的
> 首次 `is_complete=false` 落在同一个 50ms UI poll 间隔内，UI 不会看到 A 完成，导致吞句。
> 携带 prev 后，overlay 可直接读取上一句，不依赖逐帧状态转换。

### 5.2 翻译节点（dora-qwen3-translator）核心逻辑

文件：`node-hub/dora-qwen3-translator/src/main.rs`

- 从 `~/.OminiX/models/qwen3-8b-4bit`（或 `QWEN3_TRANSLATOR_MODEL_PATH` 覆盖）加载模型
- 接收 `text` 输入（ASR 转写），写入语句缓冲区
- **VAD 分句**：语言感知静音阈值（ZH=500ms、EN=350ms、FR=400ms），强制 8s 超时
- **中文句法完整性检测**：`is_syntactically_complete()` 检测 `的/地/得` 结尾或连词结尾，判断是否为残句
- 构建 prompt：系统提示含 `/no_think`（禁用 Qwen3 思考模式），翻译指令为单次、无解释
- 流式生成：每 5 个 token 发一批，`session_status=streaming`；最终完整文本，`session_status=complete`
- **Deferred question_ended**：`question_ended` 信号常早于 ASR text 到达（ASR 需处理时间）。
  translator 维护 `deferred_ended_qid: Option<i64>`，若 `question_ended` 到达时无 pending session
  则暂存 question_id，后续 ASR text 创建 session 后检查匹配 → 立即 finalize，避免 3.2s idle timeout

### 5.3 TranslationListenerBridge（动态 Dora 节点）

文件：`moxin-dora-bridge/src/widgets/translation_listener.rs`

- 完全按照 `AsrListenerBridge` 的模式实现
- 监听两个输入：`source_text` 和 `translation`
- 本地维护 `current_source_text` 和 `accumulated_translation` 缓冲
- 每次收到 `translation` chunk 时，把累积文本写入 `SharedDoraState.translation`
- 收到 `is_complete=true` 后，清空本地缓冲，等待下一句

### 5.4 TranslationOverlay Widget

文件：`moxin-widgets/src/translation_overlay.rs`

布局（深色主题）：
```
┌────────────────────────────────┐
│  ZH → EN              ● LISTENING │  toolbar (44px)
├────────────────────────────────┤
│  [原文，12px，灰色]              │  source_area
├────────────────────────────────┤
│  [译文，18px，白色，可滚动]       │  translation_scroll
└────────────────────────────────┘
```

关键 API：
```rust
overlay.set_translation(cx, &source_text, &translation, is_complete);
overlay.clear(cx);
overlay.set_lang_pair(cx, "ZH", "EN");
```

状态机：`Listening → Translating → Complete → Listening`

`#[derive(Widget)]` 自动生成 `TranslationOverlayRef` 类型。在 `app.rs` 中通过以下方式调用：
```rust
let overlay_ref = self.translation_ui.widget(id!(body.translation_overlay));
if let Some(mut overlay) = overlay_ref.borrow_mut::<TranslationOverlay>() {
    overlay.set_translation(cx, ...);
}
```

**核心方法**：
- `start_translation_dataflow(cx)` — 渲染 yml 模板，`dora start`，切换到运行态，启动 1s metrics timer
- `stop_translation_dataflow(cx)` — `dora stop`，切换回设置态，停止 timer
- `show_translation_running_panel(cx, running)` — 控制两个面板的 `set_visible`
- `add_translation_log(cx, line)` — 带 HH:MM:SS 时间戳追加日志，上限 200 行
- `poll_translation_metrics(cx)` — `ps` 读取 CPU/MEM，从 `SharedDoraState.translation.read()` 更新 VAD 状态，`read_if_dirty()` 镜像新翻译事件到日志
- `update_translation_lang_buttons(cx)` — 按当前 `translation_src_lang`/`translation_tgt_lang` 更新 8 个语言按钮的 `active` instance

### 5.8 2026-03-25 联调结论（新增）

本次 macOS 联调已定位并修复以下关键问题：

1. **翻译 dataflow 启停方式修复**
   - 由 `Command::new("dora")` 直接启动，改为使用独立 `DoraIntegration` 实例管理翻译流。
   - 修复了错误的停止命令（此前把节点名当成 dataflow 名传给 `dora stop`）。
   - 结果：`moxin-mic-input` / `moxin-translation-listener` 动态节点可稳定连接。

2. **浮窗状态源统一**
   - `app.rs` 改为从 `TTSScreen` 暴露的翻译共享状态读取，不再读取另一份脱节状态。
   - 结果：`translation_window_visible` 与翻译内容更新在同一状态源上流转。

3. **Makepad 二窗口生命周期适配**
   - 确认 `<Window>` 在初始化阶段即创建 OS 窗口（不是点击按钮后才创建）。
   - 对系统关闭按钮做拦截（转最小化）避免窗口销毁后无法重新唤起。
   - 显隐改为 OS 级最小化/恢复策略，macOS 上用 `Deminiaturize + Normalize` 避免恢复成全屏。

4. **翻译内容显示链路修复**
   - 解决 `SharedDoraState.translation.read_if_dirty()` 被主界面先消费，导致浮窗拿不到更新的问题。
   - 主界面日志镜像改为非消费式读取 + 去重指纹。
   - 浮窗侧增加更新日志与兜底 label 直写路径。
   - 结果：浮窗可显示翻译文本。

5. **ASR 输入优化**
   - `translation.yml` 中 ASR 输入由 `moxin-mic-input/audio` 改为 `moxin-mic-input/audio_segment`（VAD 分段音频）。
   - 结果：ASR 输入更聚焦语音段，噪声影响下降。

当前实测日志要点：
- `dora logs <id> asr`：可见连续转写与耗时日志。
- `dora logs <id> translator`：可见 `zh → en` 翻译输出。
- 仍偶发 `<think>/</think>` 痕迹与质量问题，属于模型输出控制/后处理问题，后续继续优化。

## 7. 已知潜在编译问题及修复方向

### 问题 A：`qwen3-mlx` API 不兼容

`main.rs` 中使用了以下 API：
```rust
use qwen3_mlx::{load_model, ModelConfig};
use mlx_lm_utils::chat_template::apply_chat_template;
// Generate<KVCache> iterator
```

如果实际 crate 的函数签名不同，需要读取 `qwen3-mlx` 的源码（路径在 workspace `Cargo.toml` 的 `[patch]` 中）来对齐。

### 问题 B：`TranslationOverlayRef::borrow_mut()` 不存在

如果 `#[derive(Widget)]` 不自动生成 `borrow_mut()`，改用：
```rust
// 方案 B：用 apply_over 更新 Label 文本
self.translation_ui
    .widget(id!(body.translation_overlay.translation_scroll.translation_label))
    .apply_over(cx, live! { text: (&translation) });
```

### 问题 C：`live_design!` 中 `visible: false` 不被 Window 接受

Makepad Window 的初始可见性控制可能需要改为：
```rust
// 在 handle_startup 中强制隐藏
self.translation_ui.apply_over(cx, live! { visible: false });
```

### 问题 D：`screen.rs` 中 translation_page 内 button 路径错误

`translation_settings_panel` / `translation_running_panel` 内的控件路径较长，如果 Makepad 报 widget not found，逐级缩短路径确认哪一层 id 写错。调试时可在 `handle_actions` 的 button 匹配处加 `log::debug!` 确认 `clicked()` 是否触发。

### 11.2 当前实测现状（已部分修复，待验证）

以下两个核心问题已完成代码修复（commit `c84541f`），但尚需 macOS 实机验证：

1. **~~吞句/覆盖问题~~ → 已修复**
   - **根因**：`SharedDoraState.translation` 使用 `DirtyValue`（last-write-wins）。当句子 A 完成（`is_complete=true`）后句子 B 立即开始，两次 `set()` 可能落在同一个 50ms UI poll 间隔内，导致 UI 从未看到 A 的 `is_complete=true`，overlay 不会将 A 推入 `prev_source/prev_translation`，直接被 B 覆盖。
   - **修复**：扩展 `TranslationUpdate` 新增 `prev_source_text` / `prev_translation` 字段。Listener 自行跟踪已完成句子，每次 `set()` 都携带 prev。Overlay 从 update 直接读取 prev，不再依赖逐帧看到 `is_complete` 状态转换。
   - **改动文件**：`data.rs`、`translation_listener.rs`、`translation_overlay.rs`、`app.rs`

2. **~~偶发”卡在翻译中”~~ → 已修复**
   - **根因**：AEC 的 `question_ended(qid=X)` 通常在 ASR 完成转写之前就已发送（ASR 需要处理时间）。Translator 收到 `question_ended` 时若无 pending session，直接忽略（`finalize_pending_session` 对 `None` 直接 return）。之后 ASR text 到达创建 pending session，但 `question_ended` 已丢失，只能等 idle timeout（3.2s），表现为”卡在翻译中”。
   - **修复**：Translator 新增 `deferred_ended_qid: Option<i64>`。当 `question_ended` 到达时无 pending session → 存储 question_id。后续 ASR text 创建 pending session 后检查是否匹配 deferred qid → 立即 finalize，无需等待 idle timeout。
   - **改动文件**：`node-hub/dora-qwen3-translator/src/main.rs`

3. **翻译质量问题（未修复，低优先级）**
   - 仍有语义偏差/不自然输出（例：`Growth step` 类错误词），属于模型/提示词/后处理综合问题，优先级低于会话稳定性。
